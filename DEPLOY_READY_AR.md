# خطة ما قبل النشر للعملاء (Go-Live Checklist)

## 1) تجهيز البنية
- [ ] اختر استضافة تدعم Node.js وتشغيل دائم (Render / Railway / VPS / Fly).
- [ ] اربط دومين رسمي (`misrsteel.com`) مع HTTPS.
- [ ] فعّل Volume/Persistent Disk لحفظ:
  - `USERS_STORE_FILE`
  - `SITE_CONFIG_FILE`
- [ ] اضبط متغيرات البيئة من ملف `.env.example`.

## 2) الأمان قبل الإطلاق
- [ ] غيّر `ADMIN_TOKEN` لقيمة قوية جدًا (32+ حرف).
- [ ] حدّد `ALLOWED_ORIGINS` على دومينك فقط (بدون `*` في الإنتاج).
- [ ] امنع فتح صفحة `admin.html` للعامة (Basic Auth / IP allowlist / مسار داخلي).
- [ ] فعل نسخ احتياطي يومي للملفات + اختبار الاسترجاع.

## 3) التشغيل التجاري
- [ ] ربط الدفع الحقيقي (Paymob) مع webhook لتأكيد الدفع آليًا.
- [ ] تعريف حالات الطلب (new/paid/processing/shipped/delivered).
- [ ] تجهيز سياسة شحن/استرجاع/خصوصية على الموقع.
- [ ] إضافة وسائل تواصل واضحة (واتساب + هاتف + بريد).

## 4) المحتوى والمنتجات
- [ ] أدخل المنتجات من `admin.html` داخل `products` مع:
  - `id`, `nameAr`, `nameEn`, `section`, `descriptionAr`, `descriptionEn`, `priceUsd`, `image`.
- [ ] راجع الأقسام في `sections` واللافتات في `posters`.
- [ ] اختبر ظهور البيانات على الصفحات النهائية قبل الإعلان.

## 5) القياس والتحليلات
- [ ] تركيب GA4 + Meta Pixel + أحداث:
  - `view_item`, `add_to_cart`, `begin_checkout`, `purchase`.
- [ ] تفعيل UTM tracking للحملات.
- [ ] لوحة متابعة يومية: Orders, Revenue, Conversion Rate, CAC.

## 6) اختبارات إجبارية قبل Go-Live
- [ ] اختبار رحلة شراء كاملة من أول السلة حتى تأكيد الطلب.
- [ ] اختبار فشل الدفع وRetry.
- [ ] اختبار إضافة/حذف/تعديل منتج من `admin.html`.
- [ ] اختبار الاسترجاع بعد Restart (الداتا ما تضيعش).
- [ ] اختبار سرعة الصفحة على الموبايل.

## 7) Soft Launch (موصى به)
- [ ] إطلاق مبدئي على 20–50 عميل/lead لمدة 3–7 أيام.
- [ ] مراقبة الأعطال والطلبات والدفع.
- [ ] إصلاح المشاكل الحرجة.
- [ ] بعد الاستقرار: إطلاق الحملات بشكل كامل.

---

## تعريف جاهزية النشر
اعتبر الموقع جاهز للنشر التجاري لما:
1. الدفع مؤكد آليًا + الطلبات متتبعة.
2. الداتا محفوظة ومؤمنة وفيها Backup.
3. لوحة الإدارة متأمنة.
4. التحليلات شغالة من أول يوم.
5. في فريق متابعة تشغيل يومي (حتى لو شخص واحد).
